# YHarvest 全链路实现文档

> 从启动到用户发消息到 AI 返回消息的完整过程。

## 一、启动阶段

### 1.1 入口：run.py

**文件：** `run.py`

```python
runtime = Runtime.from_config_file(args.config)  # 加载 TOML 配置
asyncio.run(runtime.serve())                      # 启动异步事件循环
```

`Runtime.from_config_file()` 读取 `config.onebot11.napcat.ws_reverse.toml`，解析出：
- **适配器**：`onebot11`，模式 `ws-reverse`，监听 `ws://127.0.0.1:8082/event`
- **插件目录**：`src/yqy_bot/plugins`
- **Python 路径**：`src`
- **热重载**：开启，每秒检查

### 1.2 插件发现与加载

iamai Runtime 扫描 `src/yqy_bot/plugins/` 目录，发现两个 Plugin 类：

| 文件 | 类 | 角色 |
|------|-----|------|
| `chat.py` | `YqyChatPlugin` | 被动回复：接收消息 → AI 生成 → 回复 |
| `active_life.py` | `ActiveLifePlugin` | 主动行为：后台循环，主动联系用户 |

### 1.3 插件初始化

**chat.py `__init__()`：**
```
init_db()                         → 创建 7 张 SQLite 表（幂等）
HistoryService()                  → 聊天历史读写
MemoryService()                   → 长期记忆读写
MoodService()                     → 情绪状态管理
RelationshipService()             → 用户关系管理
BehaviorService()                 → 行为决策引擎
MemeService()                     → 情绪化表情包
PersonaService()                  → 人格状态生成
ReflectionService()               → 反思存储
SocialMemoryService()             → 社会记忆网络
RelationshipReasoningService()    → 关系推理
_msg_counter = 0                  → 消息计数器（用于反思触发）
```

**active_life.py `__init__()`：**
```
同上 12 个 Service + PokeCooldown() + ConversationStarterService()
```

## 二、消息接收阶段（被动回复）

### 2.1 消息事件到达

NapCatQQ 通过反向 WebSocket 推送 OneBot11 事件到 `ws://127.0.0.1:8082/event`，iamai 的 OneBot11 适配器解析后，将事件分发给匹配的插件。

### 2.2 入口：chat.py → free_chat()

**函数：** `YqyChatPlugin.free_chat(ctx, event)`

```python
@message_handler(priority=90)
async def free_chat(self, ctx: Context, event: Event) -> None:
```

#### 步骤 1：时间衰减
```python
self._mood_service.tick()
```
- 根据 `updated_at` 与当前时间差，自动衰减 `energy`（每1h -1），增加 `loneliness`（12h +2，24h +3）
- 参数来自 [mood.json](file:///d:/jiaodian/iamai/examples/yqy_bot/config/mood.json) → `tick` 字段

#### 步骤 2：消息解析
```python
segments = event.message.segments
text = ctx.text.strip()
has_media = any(_is_media_segment(s) for s in segments)
```
- 纯图片/表情 → `user_msg = "对方发了一张图片/表情包过来"`
- 空消息或 `/` 开头 → 跳过
- 正常文本 → `user_msg = text`

`_is_media_segment()` 通过 `data` 字段的键名（`file`/`url`/`chainCount`/`resultId`）识别图片和收藏表情。

#### 步骤 3：情绪调整
```python
self._adjust_mood_from_text(text)
```
- 收到消息 → `loneliness -3`
- 含积极关键词 → `mood +3`
- 含消极关键词 → `mood -8`
- 关键词和数值来自 [mood.json](file:///d:/jiaodian/iamai/examples/yqy_bot/config/mood.json)

#### 步骤 4：关系系统
```python
self._relation_service.get_or_create_user(user_id)      # 确保用户存在
self._relation_service.update_last_chat_time(user_id)    # 更新聊天时间
self._adjust_relation_from_text(text, user_id)           # 关键词 → 好感度
```
- 积极关键词 → `favorability +2`
- 消极关键词 → `favorability -5`
- 关键词和数值来自 [relation.json](file:///d:/jiaodian/iamai/examples/yqy_bot/config/relation.json)

## 三、AI 调用阶段

### 3.1 调用链：`_ai_reply()` → `_build_system_prompt()` → LLM

**函数：** `_ai_reply(user_message, session_id, user_id)`

```python
system_prompt = self._build_system_prompt(user_id)       # 构建完整 System Prompt
messages = [{"role": "system", "content": system_prompt}]
messages.extend(self._get_history(session_id))            # 注入聊天历史
messages.append({"role": "user", "content": user_message})

return await LLMClient(config).chat_json(messages)        # 要求返回 JSON
```

### 3.2 System Prompt 构建过程

**函数：** `_build_system_prompt(user_id)`

按顺序拼接以下段落：

| 序号 | 段落 | 数据来源 | 对应 Service |
|------|------|----------|-------------|
| 1 | 当前状态（心情/精力/孤独） | `persona_state` 表 | `MoodService.describe()` |
| 2 | 当前聊天对象（昵称/身份/好感度等） | `relationship` 表 | `RelationshipService.describe()` |
| 3 | 当前人格状态（毒舌/温柔/表达偏好） | 内存 + identity | `PersonaService.describe()` |
| 4 | 你记得的重要事实 | `memory` 表 | `MemoryService.get_memories()` |
| 5 | YHarvest的观察（反思） | `reflection` 表 | `ReflectionService.get_recent()` |
| 6 | 人物关系观察（社会记忆） | `social_memory` 表 | `SocialMemoryService.describe_user()` |
| 7 | 人设提示词 | `prompt.json` | 配置加载 |

### 3.3 聊天历史注入

**函数：** `_get_history(session_id)`

```python
self._history_service.get_recent_history(session_id, MAX_HISTORY_TURNS)
```
- 从 `chat_history` 表取最近 N 轮对话（N × 2 条消息）
- `MAX_HISTORY_TURNS` 来自 [bot.json](file:///d:/jiaodian/iamai/examples/yqy_bot/config/bot.json)（默认 15）

### 3.4 LLM 配置

```toml
# config.onebot11.napcat.ws_reverse.toml
[plugin.yqy_chat.llm]
temperature = 0.7
max_tokens = 512
```
- API Key / Base URL / Model 从 `.env` 文件通过 `load_env_file()` 加载
- 使用 `LLMConfig.from_mapping()` 自动合并 TOML 配置和环境变量

## 四、回复处理阶段

### 4.1 回复生成后：`_send_reply()`

**函数：** `_send_reply(ctx, result, session_id, user_id)`

```
LLM 返回 JSON
    ↓
_parse_result() → text, face_id, send_meme
    ↓
行为决策 → emotion 分类
    ↓
send_meme=true → MemeService.get_meme_url(emotion) → 按情绪拉斗图
    ↓
真人化发送 → TypingBehaviorService
```

#### 4.1.1 结果解析：`_parse_result(result)`
```
输入：{"text": "哈哈", "face_id": "14", "send_meme": true}
输出：("哈哈", "14", True)
```
- 兼容 dict 和 list[dict] 两种返回格式
- face_id 验证：必须是纯数字

#### 4.1.2 行为决策：`_get_behavior_action(user_id)`
```
MoodService.get_state() + RelationshipService.get_or_create_user() + MemoryService.get_memories()
    ↓
BehaviorService.decide_next_action()
    ↓
返回 {"action": "chat|meme|poke|silent", "reason": "...", "emotion": "happy|teasing|comfort|angry|excited"}
```

**情绪分类逻辑：**
| 条件 | emotion |
|------|---------|
| mood > 80 | excited |
| mood 65~80 | happy |
| mood < 25 | angry |
| energy < 25 | angry |
| favorability > 85 | comfort |
| 其他 | teasing |

#### 4.1.3 情绪化斗图：`MemeService.get_meme_url(emotion)`
根据 emotion 从 [memes.json](file:///d:/jiaodian/iamai/examples/yqy_bot/config/memes.json) 选择对应的斗图 API 端点拉取图片 URL。

| emotion | 描述 |
|---------|------|
| happy | 开心 — 哈哈哈、狗头 |
| teasing | 阴阳怪气 — 熊猫头 |
| comfort | 安慰 — 摸头、抱抱 |
| angry | 烦躁 — 怒了、翻白眼 |
| excited | 兴奋 — 芜湖、起飞 |

#### 4.1.4 真人化发送：`TypingBehaviorService.send_human_like()`
```python
await asyncio.sleep(random.uniform(1, 6))     # 打字延迟 1~6 秒

# 30% 概率引用回复
if random.random() < 0.3 and event_message:
    quote_id = event_message.message_id

# >25 字拆成两条发送
chunks = _split_text(text)
for chunk in chunks:
    msg = Message()
    if quote_id: msg.append("reply", id=quote_id)
    if meme_url: msg.append("image", file=meme_url)
    msg.append_text(chunk)
    await adapter.send_message(msg, target=user_id)
```
参数来自 [human_behavior.json](file:///d:/jiaodian/iamai/examples/yqy_bot/config/human_behavior.json)

#### 4.1.5 保存聊天历史
```python
self._history_service.append_turn(session_id, user_msg, ai_text)
```
将用户消息和 AI 回复同时写入 `chat_history` 表（同一事务）。

## 五、记忆与反思阶段

### 5.1 记忆提取

```
_send_reply() 末尾
    ↓
user_raw 过规则过滤 (_should_extract_memory)
    ↓ 规则通过
_extract_memory(user_raw) → LLM 提炼事实
    ↓ save=true
MemoryService.save_memory(user_id, content, importance)
```

**规则过滤：** `_should_extract_memory(text)`
- 长度 < `MIN_TEXT_LENGTH`（8）→ 跳过
- 纯数字 → 跳过
- 无字母 → 跳过
- 在 `SKIP_KEYWORDS` 中 → 跳过
- 含疑问词 → 跳过
- 参数来自 [memory_filter.json](file:///d:/jiaodian/iamai/examples/yqy_bot/config/memory_filter.json)

**LLM 提取提示词（`_extract_memory`）：**
- 保存：身份信息、兴趣爱好、重要事件、长期计划、家庭成员
- 不保存：日常问候、表情包、临时闲聊
- 返回：`{"save": true/false, "memory": "提炼的事实", "importance": 0.1~1.0}`

### 5.2 社交记忆提取

```
_extract_social_memory(user_raw) → LLM 提取关系
    ↓ save=true
SocialMemoryService.save_social_memory(
    subject_user, target_user, relation, content, importance
)
```

- 分析用户消息是否提到了其他人（朋友、家人、同学等）
- 关系类型：关心 / 观察 / 吐槽 / 帮助
- 例如："我妹妹最近考试压力很大" → `{target_user: "妹妹", relation: "关心", content: "妹妹最近考试压力大"}`

### 5.3 反思触发

```
每回复一条消息 → _msg_counter += 1
    达到 MESSAGES_BEFORE_REFLECTION (20)
        ↓
    _generate_reflection(user_id)
        ↓
    检查今日配额（最多 3 条）
        ↓
    收集：长期记忆 + 最近聊天记录 + 关系信息
        ↓
    LLM 提炼观察/判断 → 存入 reflection 表
        ↓
    reasoning_service.generate_reasoning(user_id)
        ↓
    关系推理：记忆 + 已有反思 → 高阶洞察 → 存入 reflection 表
```

**反思示例：**
- 输入：yqy 多次提到 Agent 开发
- 输出：`"yqy最近对Agent开发特别感兴趣"`

**推理示例：**
- 记忆：yqy 天天开发 Agent + 反思：经常熬夜
- 推理：`"yqy最近工作强度比较大"`

## 六、主动行为阶段

### 6.1 后台循环：active_life.py

```python
class ActiveLifePlugin.startup():
    self._task = asyncio.create_task(self._loop())

async def _loop():
    while True:
        await asyncio.sleep(TICK_INTERVAL)    # 默认 600 秒（10 分钟）
        await self._tick()
```

### 6.2 触发检查：`_tick()`

```
loneliness > 70? ──否──→ 跳过
    ↓ 是
energy > 30? ────否──→ 跳过（日志记录）
    ↓ 是
_find_best_target() → 好感度最高用户
    ↓
_is_cooldown()? ──是──→ 跳过（4 小时内主动联系过）
    ↓ 否
BehaviorService.decide_next_action() → 行为决策
    ↓
执行：chat / meme / poke
    ↓
消耗：energy -2, loneliness -5
```

### 6.3 主动聊天生成：`ConversationStarterService.generate()`

```
输入：
- 长期记忆 (memories)
- 社交记忆 (social_memories)
- 反思 (reflections)
- 最近聊天 (history)
- 人格倾向 (identity → persona_hint)
    ↓
构建 Prompt（必须结合上下文，禁止生成"在吗""干嘛呢"等）
    ↓
LLM 生成一句话（不超过 20 字）
    ↓
后置过滤：含禁止短语则丢弃
```

### 6.4 主动表情包

```python
emotion = BehaviorService.classify_emotion(state)
meme_url = await MemeService.get_meme_url(emotion)
```

### 6.5 主动戳一戳

```python
if not PokeCooldown.can_poke(user_id):  # 24h 内戳过 → 跳过
    return
await call_api("send_poke", user_id=user_id)
PokeCooldown.record_poke(user_id)
```

## 七、状态变更汇总

每次用户消息回复后的状态变更：

| 属性 | 方向 | 数值 | 来源 |
|------|------|------|------|
| loneliness | ↓ | -3 | 收到消息 |
| mood | ↑/↓ | +3 或 -8 | 关键词触发 |
| favorability | ↑/↓ | +2 或 -5 | 关键词触发 |
| energy | ↓ | -1 | 每次回复 |
| intimacy | ↑ | +1 | 每次回复 |
| energy | ↓ | 每1h -1 | 时间衰减 |
| loneliness | ↑ | 每12h +2 | 时间衰减 |

## 八、完整流程图

```
QQ 用户发消息
    │
    ▼
NapCatQQ → WebSocket → iamai OneBot11 Adapter
    │
    ▼
chat.py free_chat(ctx, event)
    │
    ├─ 1. 时间衰减 (tick)
    ├─ 2. 消息解析 (文本/图片/表情)
    ├─ 3. 情绪调整 (关键词 → mood/loneliness)
    ├─ 4. 关系调整 (身份识别 + 好感度)
    │
    ├─ 5. 构建 System Prompt
    │      ├─ 情绪状态
    │      ├─ 关系上下文
    │      ├─ 人格状态 (毒舌/温柔/表达偏好)
    │      ├─ 长期记忆
    │      ├─ 反思观察
    │      ├─ 社会记忆
    │      └─ 人设提示词
    │
    ├─ 6. 注入聊天历史 (最近15轮)
    │
    ├─ 7. LLM API 调用 (DeepSeek V4 Flash)
    │      返回 JSON: {text, face_id, send_meme}
    │
    ├─ 8. 行为决策 → emotion 分类
    │
    ├─ 9. 按情绪拉取斗图 (MemeService)
    │
    ├─ 10. 真人化发送
    │       ├─ 随机延迟 1~6s
    │       ├─ 30% 引用回复
    │       └─ >25字 分拆发送
    │
    ├─ 11. 记忆提取 (事实 + 社交关系)
    │
    ├─ 12. 状态更新 (energy/intimacy)
    │
    └─ 13. 消息计数 → 达标触发反思 + 关系推理
             ↓
        每20条 → 反思观察 → 高阶推理 → 存入 reflection 表
```
