# YHarvest（星星）项目架构说明

## 一、项目概览

YHarvest 是一个基于 [iamai](https://github.com/retrofor/iamai) 框架 + NapCatQQ 的 QQ 个人聊天机器人，接入 DeepSeek V4 Flash 大模型。具备情绪系统、关系系统、长期记忆、反思推理、主动行为等高级人格化功能。

| 项目 | 说明 |
|------|------|
| 语言 | Python >= 3.10 |
| 框架 | iamai（异步插件式机器人框架） |
| 模型 | DeepSeek V4 Flash（OpenAI 兼容 API） |
| 协议 | OneBot11 反向 WebSocket（NapCatQQ） |
| 存储 | SQLite（WAL 模式） |
| 包管理 | uv |

## 二、目录结构

│   ├── reflection.json              # 反思系统参数

│   ├── memes.json                   # 表情包分类 API 地址
│   ├── human_behavior.json          # 真人行为模拟参数
│   └── memory_filter.json           # 记忆过滤规则
│
├── services/                        # 业务逻辑层（15个模块）
│   ├── db.py                        # 数据库连接管理 + 建表
│   ├── history.py                   # 聊天历史持久化
│   ├── memory.py                    # 长期记忆（事实提取）
│   ├── memory_filter.py             # 记忆过滤规则
│   ├── mood.py                      # 情绪系统（mood/energy/loneliness）
│   ├── relation.py                  # 关系系统（好感度/亲密度/信任度）
│   ├── persona.py                   # 人格系统（身份差异化）
│   ├── behavior.py                  # 行为决策引擎

│   └── active_life.py               # 主动行为插件（后台循环）

│
├── run.py                           # 启动入口

├── config.onebot11.napcat.ws_reverse.toml  # QQ 模式配置
├── config.terminal.toml             # 终端测试模式配置

## 四、数据库表结构（7张表）

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| `chat_history` | 聊天历史 | session_id, role, content |
| `memory` | 长期记忆 | user_id, content, importance |
| `persona_state` | 情绪状态（单行） | mood, energy, loneliness |
| `relationship` | 用户关系 | user_id, favorability, intimacy, trust |
| `behavior_log` | 行为日志 | user_id, action |
| `reflection` | 反思/推理 | content, importance |
| `social_memory` | 社会关系网络 | subject_user, target_user, relation, content |

## 五、核心系统交互图

```
用户消息 → chat.py free_chat()
    │
    ├─ tick() 时间衰减
    ├─ _adjust_mood_from_text()    关键词 → 情绪调整
    ├─ _adjust_relation_from_text() 关键词 → 好感度调整
    ├─ get_or_create_user()        确保用户记录存在
    │
    ├─ _build_system_prompt()
    │   ┌─────────────────────────────┐
    │   │ 情绪状态 (MoodService)       │
    │   │ 关系上下文 (RelationService)  │
    │   │ 人格状态 (PersonaService)    │
    │   │ 长期记忆 (MemoryService)     │
    │   │ 反思观察 (ReflectionService) │
    │   │ 社会记忆 (SocialMemorySvc)   │
    │   │ 人设提示词 (prompt.json)     │
    │   └─────────────────────────────┘
    │
    ├─ LLM 调用 → JSON 回复 {text, face_id, send_meme}
    │
    ├─ BehaviorService 决策 → emotion 分类
    ├─ MemeService.get_meme_url(emotion)    情绪化斗图
    ├─ TypingBehaviorService.send_human_like()  真人化发送
    │   ├─ 随机延迟 1~6s
    │   ├─ 30% 概率引用回复
    │   └─ >25字分拆发送
    │
    ├─ 记忆提取 (_extract_memory)
    ├─ 社交记忆提取 (_extract_social_memory)
    │
    └─ 每20条消息 → 反思 + 关系推理
```

## 六、配置驱动架构

所有业务参数集中在 `config/` 目录的 JSON 文件中，对应的 Python 模块在模块级别加载配置为模块级常量。修改参数无需改动代码，只需编辑 JSON 文件后重启 bot。

```
config/xxx.json  ──加载──→  services/xxx.py（模块级常量）
                                    │
                                    └──→ Plugin/Service 使用常量
```

## 七、运行时流程

```
1. python run.py
2. Runtime.from_config_file(config.onebot11.napcat.ws_reverse.toml)
3. 加载 src/yqy_bot/plugins/ 下所有插件
4. chat.py.__init__() → init_db() → 创建各 Service 实例
5. active_life.py.__init__() → 同上
6. active_life.startup() → 启动后台 asyncio 循环
7. OneBot11 适配器监听 ws://127.0.0.1:8082/event
8. 等待 NapCatQQ 推送消息事件
9. hot_reload: 检测代码变化自动重载
```
